# Editor Script Menu and Cleanup Work Note

## Goal
- AddComponent popup must not expose `ScriptComponent` as a normal component.
- The Script submenu must list script types from the reflection script registry.
- Korean/English labels must use localization keys, not hardcoded UTF-8 byte strings.
- Editor cleanup must be grounded in current ownership and dependency direction.

## Findings
- `SceneView`, `Hierarchy`, and `Inspector` all use `EditorGuiDrawHelpers::DrawAddComponentMenu` or `DrawAddComponentButton`.
- `ScriptComponent` was registered as a reflected component in the `Scripting` category, so the popup listed the component type itself instead of registered user scripts.
- `CReflectionRegistry` already has script metadata APIs: `GetScriptTypeCount`, `GetScriptType`, `FindScript`, and `FindScriptByName`.
- `ImGuiUtillity.h` contained non-template RAII method bodies that do not need to stay in the header.

## Decisions
- Keep the normal component category flow, but skip `ScriptComponent`.
- Add a separate localized `Script` submenu driven by `ScriptTypeInfo`.
- Use a dedicated editor command for adding scripts so add + type assignment is one undo/redo unit.
- Keep generic ImGui utility primitives under `Engine/Editor` for now; only move implementation bodies to `.cpp`.

## Changed
- Added script registry based add menu in `EditorGuiDrawHelpers`.
- Added command types for adding a script component and changing script type.
- Made `ScriptComponent` a duplicate-capable reflected component.
- Added script menu/status localization keys.
- Moved non-template `ImGuiUtillity` method bodies from header to cpp.

## Deferred
- Full localization audit for every raw editor string is larger than this pass.
- Moving `ImItem` or wider editor framework files between Engine/Application should be done after a dependency graph pass and project file update.

## 2026-05-28 Script Convenience Follow-up
- Replaced the per-facade `Engine::Debug` style API direction with a public `EngineCore` service bundle.
- Host engine fills the global `Engine` bundle from owned services; GameScript DLL copies it through `GameModuleContext::HostEngine`.
- `BindEngineCore()` also routes DLL-local `Log::*` calls to the host logger, so both `Engine.Debug->Log()` and legacy `Log::Info()` reach the editor log.
- `GameScript` now depends on `Engine` in the solution so SDK headers/libs are staged before script DLL compilation.
- `Engine/EngineAPI.h` remains as a compatibility include that forwards to `Core/EngineCore.h`; new scripts should include `GameFramework/Scripting/ScriptAPI.h`.

## 2026-05-28 GameScript Naming and Distribution Follow-up
- Renamed the old root sample/development script DLL project into the SDK validation sample `Samples/GameScriptSample`.
- Follow-up direction: the repository sample now lives under `Samples/GameScriptSample`; user projects are generated under `ProjectRoot/Contents/GameScript.vcxproj`.
- Script rebuild policy: `ScriptAutoRebuildEnabled` controls only automatic source-change scans on editor focus regain. Initial script module load and manual rebuild remain available regardless of the checkbox.
- Project generation source of truth: `SDK/Templates/GameScript.vcxproj.template` owns the MSBuild project shape; `CGameScriptProjectGenerator` only replaces placeholders and generates module/registry source files.
- Removed the redundant root sample `ExampleScript`; `DefaultScript` remains as the minimal SDK/live-compile smoke script.
- Project-generated user script projects now use `Contents/GameScript.vcxproj`, `GameScriptApi.h`, `GAMESCRIPT_EXPORTS`, and `GameScript.dll`.
- `Samples/GameScriptSample` is not the user's loaded project. It is a repository sample/SDK validation project; the editor generates a separate `GameScript.vcxproj` inside each loaded project's `Contents` folder.
- Distribution must include the editor executable/runtime DLLs, `SDK`, and `Localization`. A user project should only need `ProjectName.Jproject` plus `Contents/Assets` and `Contents/Scripts`; script support files under `Contents` are generated by the editor.
